## 1. 语音功能（核心差异点）

### Q1.1 语音读题（TTS）
使用什么技术方案？
- 微信同声传译插件？
- 第三方 API（百度/讯飞/阿里云）？
- Web Audio API 浏览器级合成？
- 其他？

A: 微信同声传译插件，已内置在微信小程序中，使用方便且性能较好。

### Q1.2 语音读题时是否同步显示字幕高亮？
即：读到的文字在屏幕上同步标记，方便用户跟上。

A: 是的，读题时会同步显示字幕高亮，帮助用户更好地跟上语音内容。


### Q1.3 语音回答（ASR）
用户"语音回答"的具体交互逻辑是哪种？
- **A 方案**：用户说选项字母（"A"/"B"/"C"/"D"），系统识别后选中对应选项
- **B 方案**：用户用自然语言说出答案内容，ASR 转文字后与正确答案匹配
- **C 方案**：完全口语化答题——用户自由表述，系统用 NLP 判断对错
- **D 方案**：其他（请描述）

A: A 方案

### Q1.4 语音与触控的关系
语音答题和手动点击是互斥还是并存？
- 语音模式下禁止触控？
- 两种模式随时可切换？
- 同一题内能否混合使用？

A: 语音答题和手动点击是并存的

### Q1.5 是否支持方言/口音？
如果只支持标准普通话，是否需要标注？

A: 不支持，只支持标准普通话，不需要标注

README 只列了"登录"和"获取问题列表"两个接口。
以下接口哪些需要、哪些不需要？

- [ ] 提交答题结果 / 记录对错
- [ ] 获取错题本列表

## 3. 数据同步

### Q3.1 同步方式
极速数据 API → 本地数据库，是全量覆盖还是增量更新？
- 如果极速数据某条题目内容变了，本地是更新旧记录还是删了重建？
- 题目 ID 是极速数据原生的还是本地自增的？如果本地自增，如何与上游保持映射？

A: 全量覆盖，如果题目内容变了则更新旧记录。极速数据中没有ID，所以本地使用自增ID，并通过题目内容的哈希值来判断是否需要更新。

### Q3.2 定时同步时间
每天的同步 cron 安排在什么时间执行？可否配置？

A: 默认每天凌晨 3 点执行同步，具体时间可通过配置项 `sync.cron` 自定义设置。

### Q3.3 启动同步的幂等性
服务每次重启都触发一次同步，如果频繁重启是否会重复全量拉取？是否应该加一个"距上次同步至少 N 小时才再执行"的保护？

A: 是的，服务启动时会检查距上次同步的时间，如果不足 N 小时（默认 24 小时），则跳过本次同步，以避免频繁重启导致的重复全量拉取。

### Q3.4 限频与失败处理
极速数据 API 有调用频率限制，超额后是静默跳过还是记录告警？同步失败是否自动重试？

A: 超额后会记录告警，并且同步失败会自动重试，重试间隔和次数可以通过配置项 `sync.retry_interval` 和 `sync.max_retries` 来调整。

## 4. 用户表

### Q4.1 用户表字段
至少需要包含哪些字段？
- 主键用 `openid` 还是 `unionid`？
- 是否存昵称（nickname）、头像（avatar）？
- 答题进度存在用户表的 JSON 字段里，还是独立的关联表（如 `user_progress`、`user_wrong_answers`）？

A: 主键使用 `openid`，存昵称（nickname）和头像（avatar）。申请获取用户的手机号 答题进度暂时前端本地存储，不放在用户表中，后续如果需要同步到后端再考虑独立关联表。

### Q4.2 注册时机
用户首次打开小程序时自动创建用户记录，还是需要用户主动点击"注册"？

A: 用户首次打开小程序时自动创建用户记录，无需主动点击注册。

## 5. 答题交互

### Q5.1 练习模式 vs 模拟考试模式
是否区分这两种模式？
- **练习模式**：答完一题立刻显示对错和解析
- **模拟考试**：全部答完才出分，考试过程中不提示对错

A: 是的，区分练习模式和模拟考试模式。练习模式答完一题立刻显示对错和解析，模拟考试模式全部答完才出分，考试过程中不提示对错。

### Q5.2 错题本机制
- 答错的题是否自动进错题本？
- 错题本是否需要按科目/分类筛选？
- 是否需要复习提醒/定时推送？

A: 答错的题自动进错题本，错题本可以按科目筛选，不区分分类。暂不考虑复习提醒/定时推送功能。

### Q5.3 答题顺序
- 顺序出题还是随机出题？
- 用户能否标记题目稍后回顾？

A: 默认顺序出题，用户可以标记题目稍后回顾，标记后会在错题本中有特殊标识。

## 6. 微信登录

### Q6.1 Token 管理
- 登录成功后使用什么机制保持会话（JWT / session / 自定义 token）？
- Token 存储在前端 storage 中？
- Token 过期后是否静默刷新（无需用户重新授权）？

A: 登录成功后使用 JWT 机制保持会话，Token 存储在前端 storage 中。Token 过期后会尝试静默刷新，如果刷新失败则需要用户重新授权登录。

### Q6.2 多设备
同一微信号在多个设备登录，数据是否互通？

A: 暂不支持多设备数据互通，每个设备的答题进度和错题本独立存储在本地，后续如果需要可以考虑增加云端同步功能。

## 7. 科目与分类

### Q7.1 分类体系
- 科目 1 和科目 4 的分类是各自独立的，还是共用一套分类标签？
- 分类数据是固定写死的（如"交通标志""交通法规"），还是从极速数据 API 动态拉取的？

A: 科目 1 和科目 4 的分类是各自独立的，分类数据是从极速数据 API 动态拉取的。
