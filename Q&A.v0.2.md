# Q&A — 驾考小程序需求澄清

> 版本: v0.2
> 说明: 基于 v0.1 已解答内容 + 新增 5 个延展问题。

---

## 1. 语音功能（核心差异点）

### Q1.1 语音读题（TTS）
使用什么技术方案？

A: 微信同声传译插件，已内置在微信小程序中，使用方便且性能较好。

### Q1.2 语音读题时是否同步显示字幕高亮？

A: 是的，读题时会同步显示字幕高亮，帮助用户更好地跟上语音内容。

### Q1.3 语音回答（ASR）

A: A 方案——用户说选项字母（"A"/"B"/"C"/"D"），系统识别后选中对应选项。

### Q1.4 语音与触控的关系

A: 语音答题和手动点击是并存的。

### Q1.5 是否支持方言/口音？

A: 不支持，只支持标准普通话，不需要标注。

---

## 2. 后端 API

### Q2.1 API 完整清单

A: 只在前端处理答题结果和错题本，后端不提供相关接口。后端只提供登录和获取问题列表接口。

### Q2.2 题目列表的拉取方式

A: 分页拉取 100 条，支持按科目过滤（参数名 `subject`，值为 1 或 4）。

### Q2.3 答案数据的传输

A: 随题目一起下发。

---

## 3. 数据同步

### Q3.1 同步方式

A: 全量覆盖，如果题目内容变了则更新旧记录。极速数据中没有 ID，所以本地使用自增 ID，并通过题目内容的哈希值来判断是否需要更新。

### Q3.2 定时同步时间

A: 默认每天凌晨 3 点执行同步，具体时间可通过配置项 `sync.cron` 自定义设置。

### Q3.3 启动同步的幂等性

A: 服务启动时会检查距上次同步的时间，如果不足 N 小时（默认 24 小时），则跳过本次同步。

### Q3.4 限频与失败处理

A: 超额后会记录告警，同步失败会自动重试，重试间隔和次数可通过配置项 `sync.retry_interval` 和 `sync.max_retries` 调整。

---

## 4. 用户表

### Q4.1 用户表字段

A: 主键使用 `openid`，存昵称（nickname）、头像（avatar），申请获取用户的手机号。答题进度暂时前端本地存储，后续如果需要同步到后端再考虑独立关联表。

### Q4.2 注册时机

A: 用户首次打开小程序时自动创建用户记录，无需主动点击注册。

---

## 5. 答题交互

### Q5.1 练习模式 vs 模拟考试模式

A: 区分练习模式和模拟考试模式。练习模式答完一题立刻显示对错和解析，模拟考试模式全部答完才出分。

### Q5.2 错题本机制

A: 答错的题自动进错题本，错题本可以按科目筛选，不区分分类。暂不考虑复习提醒/定时推送。

### Q5.3 答题顺序

A: 默认顺序出题，用户可以标记题目稍后回顾，标记后会在错题本中有特殊标识。

---

## 6. 微信登录

### Q6.1 Token 管理

A: 登录成功后使用 JWT 机制保持会话，Token 存储在前端 storage 中。Token 过期后会尝试静默刷新，如果刷新失败则需要用户重新授权登录。

### Q6.2 多设备

A: 暂不支持多设备数据互通，每个设备的答题进度和错题本独立存储在本地。

---

## 7. 科目与分类

### Q7.1 分类体系

A: 科目 1 和科目 4 的分类是各自独立的，分类数据从极速数据 API 动态拉取。

### Q7.2 跨科目查看

A: 不支持，用户需要切换科目才能查看对应的题目。

---

## 8. 配置文件（Viper）

### Q8.1 配置项清单

A: `mysql.dsn` 改为 Host、Port、User、Password、DBName 等分开配置。`sync.cron` 默认值为 `"0 3 * * *"`，`sync.on_startup` 默认值为 `true`。

### Q8.2 示例配置文件

A: 需要提供 `config.example.yaml`。

---

## 9. 数据库迁移

### Q9.1 迁移策略

A: 使用 GORM 的 `AutoMigrate` 自动同步表结构，后续表结构变更可以通过迁移脚本来管理。

## 10. 延展问题（新增）

> 以下问题由 v0.1 的解答推导而出，待你解答后并入下一版本。

### Q10 语音播报的触发方式

微信同声传译插件确定用。但具体交互：
- **语音读题**：进入题目时自动开始播报，还是用户点击"听题"按钮后才播？
- **语音回答**：ASR 是一直在听（检测到语音自动识别），还是用户按住麦克风按钮说话，松手才识别？

> 影响：决定前端录音权限申请时机和 UI 布局（是否需要一个常驻的麦克风按钮）。

A: 语音读题有开关，默认自动播报，用户可以选择关闭自动播报改为点击"听题"按钮。 读题完毕后，ASR 开始监听用户的语音输入，监听期间会有一个麦克风图标显示正在听，用户说完后自动识别，无需按住按钮。

### Q11 模拟考试模式下的答案保护

答案随题目一起下发（Q2.3），那模拟考试时：
- 前端自己隐藏答案字段，直到交卷才显示？
- 还是考试模式下调用的接口后端不返回答案字段？

> 影响：如果是前端硬藏，懂抓包的人可以提前看到答案。这个风险在意不在意？

A: 前端自己隐藏答案字段，直到交卷才显示

### Q12 题目类型

科目一 / 科目四分别有哪些题型？
- 判断题（对/错）？
- 单选题（A/B/C/D）？
- 多选题？
- 带图片的题目（交通标志、路况图）？

> 影响：决定前端题目卡片组件的设计——多选和图片支持需要提前规划。

A: 科目一和科目四的题型主要包括判断题、单选题和多选题，部分题目可能包含图片。

### Q13 前端技术选型

前端是用微信小程序原生开发（`.wxml` / `.wxss` / `.js` / `app.json`），还是 uniapp / Taro 等跨端框架？

> 影响：如果是原生，目录结构和构建方式跟跨端框架完全不同，项目脚手架一开始就要定。

A: 微信小程序原生开发。

### Q14 极速数据 API 的前置信息

后端同步功能依赖这个外部 API，需要确认：
- 接口地址和请求方式（GET/POST？）
- 请求参数（API Key 传哪里？）
- 返回的数据结构（字段名、是否有图片 URL、是否有答案解析）
- 单次请求返回多少条？是否支持分页？

> 影响：如果上游不支持分页或每次返回条数很少，全量覆盖策略需要调整（可能触发限频）。

A: 接口地址：https://api.jisuapi.com/driverexam/query
返回格式：JSON
请求方法：GET POST
请求示例：https://api.jisuapi.com/driverexam/query?appkey=yourappkey&type=C1&subject=1&pagesize=10&pagenum=1&sort=normal

type	string	是	题目类型 分为A1,A3,B1,A2,B2,C1,C2,C3,D,E,F 默认C1

pagesize	string	否	每页数量 默认1 最大值100
pagenum	string	否	当前页数
sort	string	否	排序方式 正常排序normal 随机排序rand 默认normal
chapter	string	否	章节 科目一为1-4 科目四为(41-47) 摩托车科目一为（7）科目四为（49）
