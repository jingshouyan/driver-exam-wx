## 主要逻辑

### 1. 数据同步

服务启动时 + 每日定时（cron）从极速数据 API 拉取全部题目：

1. 分页请求，每页 100 条
2. 每道题标准化处理：
   - 去掉选项前缀 `A、` → `工作证`
   - 判断题自动生成 `option1=对, option2=错`, `answer → A/B`
   - 图片 URL 按 MD5 去重，base64 存 `images` 表
3. `content_hash` 去重插入，已存在的自动更新
4. 失败自动重试（指数退避，最多 3 次）

### 2. 前端缓存与版本控制

| 步骤 | 说明 |
|---|---|
| 缓存检查 | `storage.hasCache(subject)` 判读本地是否有题目数据 |
| 版本对比 | `getQuestionsVersion` 获取服务端最新时间戳 |
| 增量更新 | 版本不一致时自动分页拉取全部题目 |
| 图片懒加载 | 切换题目时按需请求 `/images/by-url`，缓存到 storage |
| 练习进度 | `saveProgress / getProgress` 记录当前题目索引 |

### 3. 练习模式

```
加载缓存 → 恢复上次进度 → 答题 → 对错着色 → 答对自动下一题
                                              ← 左右滑动切换
```

- 答对：绿色边框 + 600ms 自动下一题
- 答错：红色边框 + 显示正确答案（绿色揭示）+ 记录错题
- 左滑下一题，右滑上一题
- 完成页展示错题回顾（含选项对错样式）

### 4. 考试模式

```
加载缓存 → 随机打乱取 100 题 → 逐题作答 → 自动下一题 → 交卷评分
```

- 从本地缓存随机抽题，每次考试题目不同
- 选完答案 500ms 自动进入下一题
- 交卷后展示分数 + 错题列表（含选项着色反馈）

### 5. 错题本

- 练习/考试中答错的题目自动加入
- 支持标记回顾、按科目筛选、清空
- 显示正确答案和对应选项文本（如 `A、对`）

## 技术栈

| 层 | 技术 |
|---|---|
| **前端** | 微信小程序原生（WXML + WXSS + JS） |
| **后端** | Go 1.23 + Gin + GORM |
| **数据库** | MySQL 8.0 |
| **部署** | 微信云托管 |
| **数据源** | [极速数据 - 驾考题库 API](https://www.jisuapi.com/api/driverexam/) |
| **AI 编程** | [Reasonix Code](https://reasonix.ai) + DeepSeek |

### 数据库表

| 表 | 用途 | 核心字段 |
|---|---|---|
| `questions` | 题目主表 | subject, question, option1-4, answer, content_hash(unique) |
| `images` | 图片缓存 | url_md5(unique), url, data(base64) |
| `users` | 微信用户 | openid, nickname, avatar |
| `sync_records` | 同步日志 | status, message, created_at |

## AI 编程实践

本项目完全通过 [Reasonix Code](https://reasonix.ai) 以对话方式编写，人机协作流程：

### 工作流

```
用户需求 → AI 理解 → 搜索代码 → 生成修改 → 提交 → 验证反馈 → 迭代
```

### 典型对话示例

```
用户: 数据同步时，将选项前面的 A、去掉，若是判断题，则 option1 对，option2 错
AI:  [搜索 sync.go → 添加 cleanOption → 更新 convertToQuestion]
     → 提交 commit

用户: 练习模式 点击后选项着色 没有变化
AI:  [调试 → 发现 WXML 函数调用问题 → 改为预计算 optClass]
     → 提交 commit

用户: 图片数据太大，拆分到独立表，懒加载并缓存
AI:  [设计 Image 模型 → 重构 sync → 添加 /images/by-url 端点
     → 前端 lazy load + 缓存] → 提交 commit
```

### 关键设计决策（AI 辅助）

| 决策 | 原因 |
|---|---|
| 图片独立表 + URL MD5 去重 | 避免分页同步超限，图片复用 |
| 前端全量缓存 + 版本对比 | 离线可用，减少云托管请求次数 |
| 预计算 optClass 而非 WXML 函数调用 | 微信小程序 WXML 函数调用的兼容性问题 |
| Fisher-Yates 洗牌随机抽题 | 保证公平且每次考试不重复 |
| fsnotify + CRON 定时同步 | 启动时同步 + 每日增量更新 |

## 本地开发

### 后端

```bash
cd backend
cp config.example.yaml config.yaml   # 编辑 API Key
go run ./cmd/server
```

### 前端

用微信开发者工具打开 `frontend/` 目录。

### 配置切换

`frontend/config.js`：

```js
useCloud: false,              // 本地开发
apiBaseUrl: 'http://localhost:8080'

useCloud: true,               // 云托管部署
cloudEnv: '你的环境ID',
cloudService: '你的服务名',
```

## 部署

1. 云托管控制台创建服务，上传 Dockerfile 构建
2. 配置环境变量和 MySQL 数据库
3. 小程序管理后台配置云托管域名
4. `frontend/config.js` 填写 `cloudEnv` 和 `cloudService`
5. 上传小程序代码并提交审核
